▶️ EdgeLake Demo – Quick Start

A minimal, operator‑friendly guide to launching and exploring the EdgeLake data fabric.

OVA boots → containers auto‑start GUI, Query, Operator x2 Local‑only endpoints

Automatic Start

When the OVA boots, all required containers are launched automatically.

No action needed for a first‑run demo.

Manual Start (if needed)

Launch all EdgeLake containers with:

cd ~/Edgelake
./ELstartup.sh

This starts:

  • gui-1 – Web UI for monitoring and data management
  • edgelake-demo-master – Control plane
  • edgelake-demo-query – Query node (SQL over Edgelake)
  • edgelake-demo-operator – Data operator node
  • edgelake-demo-operator2 – Second data operator node

Service Endpoints

ServiceRoleEndpoint
GUI gui-1 Web UI http://localhost:3001
Query Node REST edgelake-demo-query SQL & Fabric Query vmipaddr:32349
Operator Node REST edgelake-demo-operator Data Ingest / Control vmmipaddr:32149
Operator2 Node REST edgelake-demo-operator2 Data Ingest / Control vmipaddr:32159
Using the GUI with a node

Open the GUI and insert the VM_IP:Port of a Query or Operator node in the Connection dialog, then click Use.

Monitor

Track resources and health across nodes from the GUI dashboard.

Add Data

Load JSON‑formatted data into an operator node via the GUI.

SQL Query

Build and run SQL queries from the query node’s GUI tool.

Blockchain Manager

Inspect blockchain metadata that underpins the data fabric.

Default Demo Dataset

  • Database: new_company
  • Table: rand_data
This table is pre‑populated via a cloud MQTT feed on first launch.

Add Your Own Data

  1. Use the GUI (Operator node)
    Click Add Data and provide JSON‑formatted input.
  2. Subscribe to a MQTT broker
    MQTT Background Services (docs)
  3. Insert via REST API
    REST Insert Examples
Tip: Ensure you target an operator node for data insertion and the query node for read/SQL workflows.
